# Landing Page Spec

## Purpose

The landing page has one job: make delegated AI action feel valuable and trustworthy before the user ever sees the app.

## Route

`/`

## Primary CTA

`Enter The World`

Secondary CTA:

`See How It Works`

## Core Message

Headline direction:

"Authorize your AI agent. Watch it act with limits, not guesswork."

Supporting line:

"Actify AI turns delegated commerce into a visible world of permissions, missions, and auditable actions."

## Layout Sections

### 1. Hero

- Large statement on the left
- Stylized dashboard or world preview on the right
- CTA row
- Small trust strip with Auth0, AI, and realtime hints

### 2. Why This Matters

- Explain the problem with invisible AI agents
- Explain how Actify AI adds visibility, permissioning, and control

### 3. How It Works

Three-step flow:

1. Sign in and define limits
2. Send the agent into the world
3. Approve, observe, and audit every action

### 4. Security Story

- Delegated permissions
- Step-up confirmation
- Action logs
- No blind execution

### 5. World Preview

- Show the three-pane dashboard concept
- Left rail labels
- Center world frame
- Right detail pane

### 6. Final CTA

- Strong repeat CTA to sign in
- Optional note that the app is optimized for desktop and landscape use

## Visual Direction

- Navy and black atmospheric background
- Cyan as the primary technical accent
- Sharp panel edges for surfaces
- Rounded buttons for actions
- Bold display typography with condensed headings
- Motion should feel purposeful, like systems booting or panels activating

## Suggested Copy Blocks

### Trust Pillars

- `Permissioned by design`
- `Observable in real time`
- `Built for agent action`

### Feature Cards

- `Agent Missions`
- `World Navigation`
- `Approval Vault`
- `Live Decision Feed`

## Interaction Notes

- The right side of the hero preview should lightly animate as if the world is active
- CTA click should transition into login, not a dead marketing page
- If auth is unavailable, show a graceful waitlist or disabled state, not a broken button

## Responsive Rules

- Desktop first
- Tablet keeps the two-column hero
- Mobile stacks content vertically
- Do not try to show a full playable world on the landing page

## Definition Of Done

- Clear value in the first viewport
- Strong visual preview of the actual app
- CTA is wired to auth
- Copy speaks to both security and agent action
